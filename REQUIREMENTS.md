# video-clone 要件定義書

最終更新：2026-04-29

---

## 1. 概要

参考動画のURLを入力するだけで、構成・スタイル・テロップ・音声を真似た動画を全自動で生成するシステム。

社内で動画を作成して縦型動画プラットフォームに投稿する用途。

---

## 2. 背景・きっかけ

- MakeAI_CEO mana氏のXポストで「Claude Code丸投げで参考動画を再現できる」という主張がバズった
- 元記事：https://x.com/MakeAI_CEO/status/2045357049060753490
- 会社からこのシステム構築を依頼された
- ユースケースは未定だが、社内動画制作の自動化が目的

---

## 3. 機能要件

### 3.1 入力
- 参考動画のURL（または動画ファイル）

### 3.2 出力
- 動画ファイル（縦型 9:16）
- 投稿は手動（システムは生成のみ）

### 3.3 真似る要素（参考動画から完全コピー）
- 構成（カット割り・テンポ・尺）
- スタイル・雰囲気（撮り方・色味・編集の感じ）
- テーマ・ジャンル
- キャラの感じ（年齢・性別・属性）
- テロップ（元動画と全く同じ内容）
- 音声・ナレーション（元動画と全く同じ内容）

### 3.4 自動化レベル
- **完全自動（パターンA）**
- URL入れたらボタン1つで動画ファイル完成
- 失敗動画もそのまま出力（量産前提）
- 想定成功率：約2.8%（10カット繋いで全成功する確率）
- 100本生成して使えるの数本拾う運用

### 3.5 ブランディング対応
- 固定キャラで量産（ブランド化）
- 動画ごとに別キャラ
- 両方対応できる構造

---

## 4. 非機能要件

### 4.1 公開先
- 縦型動画プラットフォーム（Instagram Reels / TikTok / YouTube Shorts など）
- アスペクト比：9:16

### 4.2 品質
- 元ポストの動画と同じ品質を目指す
- ただしパターンA（完全自動）を選択しているため、実際のクオリティは「拾える数本」になる前提

### 4.3 ユーザーインターフェース
- CLI（最初は）
- 楽な構築を優先

### 4.4 量
- 未定
- 量産前提の設計にする

### 4.5 予算
- 一旦特になし

## 5. 技術選定

### 5.1 役割と候補ツール

| 役割 | 候補 | 備考 |
|---|---|---|
| 参考動画解析 | Gemini Vision / GPT-4 Vision | 構成・テロップ・キャラ抽出 |
| 音声書き起こし | Whisper API | 元動画のセリフ取得 |
| 画像生成 | GPT Image 2.0 | キャラ静止画 |
| 動画生成 | **Seedance 1.5 Pro**（fal.ai経由）| image-to-video、5秒$0.26、100本$26 |
| 音声合成 | ElevenLabs / OpenAI TTS | ナレーション |
| API窓口 | fal.ai | 各種AI統一インターフェース |
| 編集 | FFmpeg / Hyperframes | カット繋ぎ・テロップ・音声 |
| 言語 | Python / Node.js | バックエンド |
| 実行環境 | CLI | 楽な構築優先 |

### 5.2 主要ツール選定理由

- **fal.ai**：複数の動画生成AIを統一APIで使える。直接APIだと中国系（Kling・Seedance）のアカウント登録が大変
- **GPT Image 2.0**：人物画像のクオリティが安定
- **Whisper**：日本語含めた多言語の書き起こし精度が高い

### 5.3 検討中

- 編集パイプライン（FFmpegスクリプト直書き vs Hyperframes経由）

### 5.4 確定済み

- **動画生成AI：Seedance 1.5 Pro（fal.ai経由）** — 2026-04-29 決定
  - 理由：image-to-videoのキャラ一貫性が強い、最安（5秒$0.26）、量産前提と相性◎
  - 比較：Kling 2.5 $0.35、Veo 3.1 $0.50〜$1.00、Sora 2 Pro $2.50

- **キャラ一貫性：B（ゆるく頑張る）** — 2026-04-29 決定
  - GPT Image 2.0 で1枚ベース画像を作成 → 各カットで「同じキャラ・違うポーズ」をプロンプトで指示
  - 完璧は目指さない、ストーリー成立する程度の一貫性で十分
  - 後で C（Reference Image / LoRA）に昇格可能な構造にしておく

- **失敗動画の判定：A（全部出す）** — 2026-04-29 決定
  - 自動判定なし、すべて output に出力
  - 人間が目視で拾う運用
  - 1回実行=1本生成（バッチ機能なし）、100本欲しければ100回回す
  - 必要になったらC（明らかな失敗だけ自動除外）に昇格可能

- **量の目標値：月30本生成** — 2026-04-29 決定
  - 想定コスト：Seedance動画生成のみで約$78（約12,000円）/月
  - 周辺API（Vision/Whisper/TTS/GPT Image）込みで約$100/月想定
  - 成功率2.8%なら投稿可能本数は月1本程度
  - 仮の数値、運用開始後に再調整

- **BGM：B（フリー素材から選択）** — 2026-04-29 決定
  - 候補：DOVA-SYNDROME / 甘茶の音楽工房 / YouTube Audio Library 等
  - 元動画の雰囲気タグ（テンション高め/しっとり/コミカル等）でマッチング
  - 元動画のBGM完コピは著作権リスク高すぎるため不採用
  - 「全く同じ」方針はテロップ・セリフまで。BGMはフリー素材で代替

## 6. システム構成

### 6.1 処理フロー

```
[参考動画URL]
    ↓
[Step 1] 動画ダウンロード（yt-dlp など）
    ↓
[Step 2] 動画解析
    ├─ Vision AI → 構成・カット割り・キャラ属性
    └─ Whisper → セリフ・タイミング
    ↓
[Step 3] キャラ画像生成（GPT Image 2.0）
    ├─ 一貫性のため初期画像から複数バリエーション
    ↓
[Step 4] 各カット動画化（fal.ai 経由で動画生成AI）
    ├─ image-to-video × カット数
    ↓
[Step 5] 音声合成（TTS）
    ├─ 元動画と同じセリフでナレーション生成
    ↓
[Step 6] 編集・繋ぎ（FFmpeg / Hyperframes）
    ├─ カット繋ぎ
    ├─ テロップ配置
    ├─ 音声合成
    ├─ BGM（必要なら）
    ↓
[出力] 動画ファイル（mp4, 9:16）
```

### 6.2 ディレクトリ構成

```
video-clone/
├── REQUIREMENTS.md
├── HANDOVER.md
├── CLAUDE.md
├── pyproject.toml
├── uv.lock
├── main.py                       # CLI エントリポイント（typer 登録のみ）
├── src/
│   ├── __init__.py
│   ├── ffmpeg_utils.py           # ffmpeg ヘルパー（duration / フレーム抽出）
│   ├── download.py               # Step 1: yt-dlp ダウンロード
│   ├── transcribe.py             # Step 2-1: Whisper 書き起こし
│   ├── analyze.py                # Step 2-2: GPT-4o Vision 構成解析
│   ├── generate_character.py     # Step 3: GPT Image キャラ生成
│   ├── generate_clip.py          # Step 4: fal.ai Seedance 動画生成
│   ├── synthesize.py             # Step 5: OpenAI TTS 音声合成
│   ├── compose.py                # Step 6: HyperFrames 合成
│   └── clone.py                  # 全工程統合コマンド
├── cache/                        # 中間生成物（gitignore）
├── output/                       # 生成された動画（gitignore）
└── .env                          # API キー（gitignore）
```

## 7. リスク・注意事項

### 7.1 著作権・肖像権リスク

**ステータス：オーナーが「特になし」と判断（2026-04-29）**

- 参考動画のテロップ・セリフをそのまま使うため、著作権侵害の可能性
- AI生成キャラとはいえ、参考動画と酷似していれば肖像権の問題も
- 訴訟リスクは認識のうえで進行

### 7.2 プラットフォーム規約リスク

- TikTok・Instagram・YouTube Shorts ともに「丸パクリ動画」はBAN対象
- AI生成コンテンツの開示義務（プラットフォームごとに異なる）
- 違反時、会社アカウントが停止される可能性

### 7.3 技術的リスク

- 完全自動の成功率が低い（約2.8%）
- 生成失敗時のエラーハンドリング設計が必要
- 動画生成APIのコストが嵩む可能性
- 同じキャラの一貫性が保てない問題（カットごとに顔が変わる）

### 7.4 緩和策

- 投稿は手動なので、最終チェックで人間が判断できる
- 後で「AI生成テロップに切替」できる構造にしておく（将来の方針転換を許容）
- 本番運用前にリスクをもう一度警告する
